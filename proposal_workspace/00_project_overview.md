# 00. 프로젝트 개요 (Project Overview)

## 1) 문서 목적
본 워크벤치는 공공기관 용역 공고문/RFP를 입력으로 받아, 사업계획서(제안서) 작성 작업을 **절(Section) 단위 작업카드**로 분해하고, 각 절을 **A4 0.5~1쪽 분량**으로 일관되게 생성·검토·병합하기 위한 Markdown 기반 운영 템플릿이다.

## 2) 적용 범위
- 공공기관 정보화/컨설팅/운영/유지관리/연구 용역 제안서 작성
- 기관별 양식 차이는 반영하되, 본 템플릿의 공통 구조를 기본으로 사용
- 본문 중심 작성(표·도식은 설명 보조)

## 3) 핵심 운영 원칙
1. **요구사항 정합성 우선**: 모든 문단은 공고문/RFP 요구사항과 직접 연결되어야 한다.
2. **절 단위 생산**: 대문서 일괄 작성이 아니라, 절별 카드 기반으로 생성한다.
3. **분량 제어**: 절별 0.5~1쪽 단위 초안을 우선 확보한다.
4. **검토-수정 루프**: 생성 후 반드시 요구사항 커버리지·표현 품질·증빙 가능성을 점검한다.
5. **추적 가능성**: 요구사항 ↔ 절 ↔ 문장 근거를 표 형태로 관리한다.

## 4) 산출물 구조
- `section_cards/`: 절별 작업카드 템플릿 및 카드 인스턴스
- `drafts/`: 절별 초안 파일
- `final/`: 병합된 최종본 및 제출본 편집 파일
- `09_rfp_extraction_template.md`: RFP 핵심정보 구조화 템플릿
- `10_outline_planning_template.md`: 목차·배점·담당자 연계 템플릿

## 5) 권장 사용자 역할
- PM/제안총괄: 요구사항 분석, 목차 확정, 최종 승인
- 도메인 리드: 기술·사업 내용 작성
- 편집/품질 담당: 문체 통일, 중복 제거, 형식 점검
