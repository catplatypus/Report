# 03. Section Type 카탈로그

> 목차의 각 절을 아래 타입 중 하나(또는 복합)로 지정하여 카드 기반 작성을 수행한다.

## 공통 타입 목록 (11개)
1. PROJECT_UNDERSTANDING (사업이해)
2. STRATEGY (추진전략)
3. TECHNICAL_METHOD (기술/수행방법)
4. PROCESS (수행절차)
5. SCHEDULE (일정/마일스톤)
6. ORG_ROLE (조직/역할)
7. DELIVERABLES (산출물)
8. QUALITY_MANAGEMENT (품질관리)
9. SECURITY_DATA_MANAGEMENT (보안/데이터관리)
10. COMMUNICATION_REPORTING (소통/보고)
11. RISK_MANAGEMENT (리스크관리)

## 운영 권장안 (과다/과소 방지)
- **기본 제안(8개 코어 타입)**: 1~8번을 기본 구성으로 사용
- **조건부 추가(3개 확장 타입)**: 9~11번은 RFP가 명시하거나 배점이 큰 경우에만 추가
- 즉, 일반 사업은 8개 중심, 보안·대외협업·리스크 배점이 높은 사업은 11개 전체 사용

## 타입 매핑 가이드
| RFP 목차/요구 키워드 | 추천 section_type | 비고 |
|---|---|---|
| 사업 이해, 추진 배경, 현황 분석 | PROJECT_UNDERSTANDING | 문제정의/목표 일치 강조 |
| 추진 방향, 차별화 방안, 실행 전략 | STRATEGY | 전략-실행 연결 필수 |
| 방법론, 기술체계, 아키텍처 | TECHNICAL_METHOD | 도식+설명 병행 |
| 단계별 수행, 상세 절차 | PROCESS | 입력-활동-출력 구조 |
| 일정, 공정, 마일스톤 | SCHEDULE | 기간·의존성 명시 |
| 인력투입, 조직도, 역할책임 | ORG_ROLE | R&R, 의사결정 체계 포함 |
| 산출물 목록·정의·제출계획 | DELIVERABLES | 검수 기준과 연계 |
| 품질보증, 시험/검토 계획 | QUALITY_MANAGEMENT | 측정 지표 필요 |
| 보안대책, 개인정보, 백업/복구 | SECURITY_DATA_MANAGEMENT | 법규 준수 명시 |
| 회의체, 보고체계, 이슈 공유 | COMMUNICATION_REPORTING | 주기·책임자·양식 |
| 위험 식별/대응/모니터링 | RISK_MANAGEMENT | 사전/사후 대응 체계 |

## 운영 규칙
- 하나의 절에는 1개 주요 타입을 지정한다.
- 필요 시 보조 타입을 최대 2개까지 부여한다.
- 타입이 불명확하면 "필수 포함 항목"이 가장 많이 겹치는 타입을 우선한다.
