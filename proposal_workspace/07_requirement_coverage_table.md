# 07. 요구사항 커버리지 테이블

> 공고문/RFP의 요구사항을 절별 카드와 연결해 누락을 방지한다.

## 1) 작성 원칙
- 요구사항은 가능한 한 원문 문장 기준으로 분해한다.
- 각 요구사항에 고유 ID를 부여한다. (예: RQ-01, RQ-02)
- 최소 1개 이상의 section card에 매핑한다.
- 배점이 높은 요구사항은 반영 위치를 2중 확인한다.

| 요구사항 ID | 요구사항 구분(필수/선택) | 공고문 원문 요약 | 출처(문서/페이지) | 평가항목/배점 | 대응 전략 요약 | 매핑 section_type | 매핑 카드 ID | 반영 위치(파일/절) | 반영 상태(미반영/초안/반영완료) | 검토자 확인 | 근거 메모 |

## 3) 품질 점검
- [ ] 모든 필수 요구사항에 ID가 부여되었는가?
- [ ] 미반영 요구사항이 남아있지 않은가?
- [ ] 배점 높은 항목이 충분한 분량/근거로 작성되었는가?
- [ ] 최종본 반영 위치가 추적 가능한가?
- [ ] 검토자 확인란이 최신 상태인가?
