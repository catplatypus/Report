# 02. 제안서 문체·작성 규칙 (Style Rules)

## 1) 목적
공공기관 용역 제안서 자동작성 시 문체 일관성, 평가 친화성, 요구사항 정합성을 확보하기 위한 규칙이다.

## 2) 필수 원칙
1. **과장 표현 금지**
   - 금지 예: "압도적", "유일무이", "완벽", "반드시 최고" 등
   - 권장: 근거 기반의 객관적 표현(수치, 실적, 절차, 책임체계)

2. **"단순 ~이 아니라" 표현 금지**
   - 금지 예: "단순 운영이 아니라 혁신을 제공"
   - 권장: "운영 안정화와 개선 과제를 병행하여"처럼 직접적·중립적 문장 사용

3. **공고문/RFP 요구사항 반영 우선**
   - 각 절의 모든 하위 항목은 요구사항 ID와 연결 가능해야 함
   - 요구 없는 확장 제안은 부록 또는 참고 수준으로 제한

4. **A4 0.5~1쪽 단위 작성 원칙**
   - 절별 초안은 한 번에 0.5~1쪽 분량으로 생성
   - 필요한 경우 절을 2~3개 소절로 쪼개서 작성

## 3) 문장/구성 규칙
- 한 문단 3~5문장 권장
- 한 문장은 가능하면 30자~80자 내외 유지
- 결론 먼저 제시 후 근거·절차·성과지표 순서로 전개
- 추상 명사 반복("고도화", "선진화") 최소화

## 4) 금지/주의 표현
- 근거 없는 최상급(최고, 최적, 완전)
- 타사/타기관 비교 비방
- 법적·계약상 확정 불가 사항의 확정적 표현
- 평가항목과 무관한 장황한 회사 홍보

## 5) 표/본문 균형
- 표는 요약, 본문은 해설 역할로 구분
- 표만 제시하고 해설 생략하지 않음
- 도식 사용 시 핵심 메시지 1~2문장 병기

## 6) 검토 체크리스트
- [ ] 문장에 과장 표현이 없는가?
- [ ] "단순 ~이 아니라" 문형을 사용하지 않았는가?
- [ ] 각 문단이 RFP 요구사항과 연결되는가?
- [ ] 절별 분량이 0.5~1쪽 범위인가?
- [ ] 표와 본문이 상호 보완적인가?
