# drafts 폴더 사용법

## 목적
절별 초안 문서를 저장하고 검토 이력을 관리하는 작업 공간이다.

## 저장 규칙
- 파일명 예시: `01_PROJECT_UNDERSTANDING_draft.md`
- 카드 ID를 파일 상단에 명시한다.
- 버전 관리가 필요하면 `v1`, `v2` 접미사를 사용한다.

## 권장 본문 구조
1. 절 제목
2. 핵심 메시지
3. 본문(0.5~1쪽)
4. 표(필요 시)
5. 요구사항 반영 근거 요약

## 검토 연계
- 검토 의견은 파일 하단 또는 별도 리뷰 노트에 기록한다.
- 반영 후 `07_requirement_coverage_table.md` 상태를 갱신한다.
