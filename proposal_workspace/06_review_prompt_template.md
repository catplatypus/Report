# 06. 절별 검토 프롬프트 템플릿

## 사용 목적
생성된 절 초안을 평가 관점에서 점검하고 수정지시를 도출한다.

## 입력 패키지
- 절 제목:
- section_type:
- 연결 요구사항 ID:
- 요구사항 원문 요약:
- 평가항목/배점:
- 해당 카드의 필수 포함 항목:
- 초안 본문:

## 검토 기준
1) 요구사항 정합성: RFP 요구와 직접 연결되는가?
2) 정보 충분성: 평가자가 판단 가능한 수준인가?
3) 표현 품질: 과장/모호/중복 표현이 없는가?
4) 형식 적합성: 표/본문 지시와 분량 기준을 지키는가?
5) 실행 가능성: 일정·인력·방법이 현실적인가?
6) 채점 친화성: 배점 항목에 대응 문장이 명시되어 있는가?

## 출력 형식
- 총평(5줄 이내):
- 주요 이슈 목록:
  - 이슈1: 문제 / 근거 / 수정 지시
  - 이슈2: 문제 / 근거 / 수정 지시
- 누락 항목 체크:
  - [ ] 필수 포함 항목1
  - [ ] 필수 포함 항목2
- 문장 교정 제안(원문→수정문):
- 수정 우선순위(상/중/하):
- 수정 후 기대 효과:
