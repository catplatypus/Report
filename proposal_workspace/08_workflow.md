# 08. 운영 워크플로우 (RFP 입력 → 최종 병합)

## 전체 단계 개요
- 시작 전 `proposal_workspace/README.md`를 읽고 공통 규칙을 확인한다.

1. RFP 입력 및 정리
2. 요구사항 분석
3. 목차(절 구조) 생성
4. 절별 작업카드 생성
5. 절별 초안 생성
6. 절별 검토·수정
7. 통합 병합 및 최종 점검

---

## 1단계) RFP 입력 및 정리
- 공고문, 제안요청서, 과업지시서, 평가기준 문서를 수집한다.
- 문서 버전/공고번호/마감일을 기록한다.
- 요구사항 파악에 필요한 핵심 문장에 표시한다.
- `09_rfp_extraction_template.md`에 핵심 정보를 먼저 구조화한다.

산출물:
- `07_requirement_coverage_table.md` 초기 행 구성

## 2단계) 요구사항 분석
- 필수요건/정량평가/정성평가 항목을 분해한다.
- 요구사항 ID(RQ-xx)를 부여한다.
- 배점이 높은 항목을 우선순위로 태깅한다.

산출물:
- 요구사항-평가항목 매핑 완료본

## 3단계) 목차(절 구조) 생성
- RFP 권장 목차가 있으면 우선 반영한다.
- 각 절에 `section_type`을 지정한다.
- 절별 목표 분량(0.5~1쪽)을 설정한다.
- `10_outline_planning_template.md`로 절/배점/담당자 계획을 확정한다.

산출물:
- 절 목록 + 타입 매핑표

## 4단계) 절별 작업카드 생성
- `04_section_card_template.md`를 복제하여 절별 카드 생성
- `section_cards/*_template.md` 전용 필드 반영
- 작성자/검토자/완료기준 지정

산출물:
- 절별 카드 세트

## 5단계) 절별 초안 생성
- `05_generation_prompt_template.md`에 카드 정보를 주입한다.
- 절별로 A4 0.5~1쪽 초안을 생성한다.
- 초안은 `drafts/`에 파일로 저장한다.

산출물:
- 절별 초안 파일

## 6단계) 절별 검토·수정
- `06_review_prompt_template.md`로 절별 품질 검토 수행
- 누락 요구사항, 과장 표현, 분량 초과/부족 수정
- 반영 결과를 커버리지 테이블 상태에 갱신

산출물:
- 검토 반영 완료 초안

## 7단계) 통합 병합 및 최종 점검
- `drafts/` 문서를 `final/` 제출본 구조로 병합
- 문체·용어·표 번호·참조 일관성 최종 점검
- 요구사항 커버리지 100% 여부 확인

산출물:
- `final/` 최종본
- 제출 직전 체크리스트 완료본

---

## 권장 운영 팁
- 병렬 작성 시에도 커버리지 테이블은 단일 기준본으로 유지한다.
- 배점 높은 절은 검토 라운드를 1회 추가한다.
- 최종 병합 전 중복 문장과 상충 표현을 반드시 제거한다.
